## 1. 项目简介

*朱码*是一款受 [Markdown](https://daringfireball.net/projects/markdown/) 和 [Showdown](https://github.com/showdownjs/showdown) 所启发的全功能型标记语言系统。*朱码*通过为纯文本内容制定一系列规范化的语法规则，使其能被轻松地解析为标准的 HTML 代码，供支持 web 页的场景呈现格式化的内容，并避免直接存储 HTML 代码可能造成的危险或麻烦。对于熟练使用 Markdown 的用户，*朱码*的语法很容易理解，但两者的区别也非常显著，完全相同的语法会在相应的章节标题中注明`（同 Markdown）`。

出于安全性考虑，*朱码*并不包含解析表单类标签的语法，除了在“任务列表”中利用 `<input type="checkbox">` 表现勾选以外，这是沿袭了 Markdown 的习惯。

## 2. 语法概览

### 2.1 关于标签

HTML 的标签主要分为块级元素和行内元素两大类，两者的主要区别在于是否占据页面的全部宽度。因此，标记语言的块级元素应占据一个或多个整行，没有任何块级元素标记的行统一解析为段落（`<p></p>`）；根据标记方式的不同，块级元素可以分为以下两类：
1. **逐行标记型**：此类语法的每一行都必须按相应规则做标记，例如标题、列表和引用块语法必须在行首做标记；表格等等标记方式更复杂一些；
2. **标记行包夹型**：此类语法必然由多行构成，并以一个起始标记行和一个终止标记行来框定范围，例如代码块由两个 ```` ``` ```` 行包夹而成。

行内元素则几乎可以出现在任何位置，通常以两侧包裹的形式标记其所属范围。

### 2.2 关于属性

HTML 的标签中会使用各种各样的属性来设置不同的参数。有些属性是全局性的，即任何标签都可以使用，例如 ` id=""`；有些属性是专用的，即某一个或几个标签才能使用的属性，例如 ` cite=""` 是 `<q>` 和 `<blockquote>` 标签专用的。专用属性又分为可选和必须两种。` cite=""` 就是可选属性，没有它也不影响 `<q>` 和 `<blockquote>` 标签的基本功能；而 ` src=""` 属性则是 `<img>` 标签的必须属性，缺少该属性，`<img>` 标签根本不能显示图片。

针对属性的不同情况，*朱码*也有不同的处理方法。全局属性在第 4 章第 2 节中单独讲解，专用属性则融入到各自对应的标签语法中。必要属性是标签语法的重要组成部分，缺少了必要属性，对应的标签语法无法成立，也就不可能解析输出为对应的标签了。可选属性也在对应的标签语法中，用 `【】`包裹以示区别。

## 3. 标签语法规则

### 3.1 标题（同 Markdown）

| 语法              | 描述                                 | 解析输出            |
| ---------------- | ------------------------------------ | ------------------ |
| `# 一级标题`      | 行首是一个井号 + 至少一个空格的行      | `<h1>一级标题</h1>` |
| `## 二级标题`     | 行首是两个连续的井号 + 至少一个空格的行 | `<h2>二级标题</h2>` |
| `### 三级标题`    | 行首是三个连续的井号 + 至少一个空格的行 | `<h3>三级标题</h3>` |
| `#### 四级标题`   | 行首是四个连续的井号 + 至少一个空格的行 | `<h4>四级标题</h4>` |
| `##### 五级标题`  | 行首是五个连续的井号 + 至少一个空格的行 | `<h5>五级标题</h5>` |
| `###### 六级标题` | 行首是六个连续的井号 + 至少一个空格的行 | `<h6>六级标题</h6>` |

---

### 3.2 文本修饰

Markdown 只有少量文本强调语法，其中“粗体”和“斜体”的前后缀还容易混淆。*朱码*重新设计的这部分语法，不仅避免了可能混淆的情况，更大量扩充了可用的语法，充分利用 HTML 的标签功能来修饰文本。

| 语法            | 描述                                       | 解析输出                        | 
| --------------- | ------------------------------------------ | ------------------------------- |
| `**粗体`**      | 被双星号所包裹的内容                         | `<strong>粗体</strong>`         |
| `%%斜体%%`      | 被双百分号所包裹的内容                       | `<em>斜体</em>`                 |
| `~~删除线~~`    | 被双波浪号所包裹的内容                       | `<del>删除线</del>`              |
| `__下划线__`    | 被双下划线所包裹的内容                       | `<ins>下划线</ins>`              |
| `!!高亮!!`      | 被双感叹号所包裹的内容                       | `<mark>高亮</mark>`              |
| `^^上标^^`      | 被双上尖号所包裹的内容                       | `<sup>上标</sup>`                |
| `<<下标>>`      | 被两对尖括号所包裹的内容                     | `<sub>下标</sub>`                |
| `@@小字体@@`    | 被双艾特号所包裹的内容                       | `<small>小字体</small>`          |
| `{{文字^音标}}` | 被两对花括号所包裹且被上尖号分隔为两部分的内容 | `<ruby>文字<rt>音标</rt></ruby>` |

 注：本节内所有语法均可组合嵌套使用，例如 `**%%粗体 + 斜体%%**` 被解析为 `<strong><em>粗体 + 斜体</em></strong>`。

### 3.3 表情

**语法**：`::emoji_name::`

**详述**：被双冒号包裹的一串能匹配到表情名称的字符串。

**示例**：`::smile::` → 😄，`::heart::` → ❤️

### 3.4 代码（同 Markdown）

#### 3.4.1 行内代码

**语法**：`` `代码` ``

**详述**：被相同数量的反引号所包裹的内容

**解析输出**：`<code>代码</code>`

#### 3.4.2 代码块

**语法**：
````
```【代码语言】
代码内容
```
````

**详述**：被两个行首是至少三个连续且数量相同的反引号的行所包夹的多行内容，起始标记行中支持在 ```` ``` ```` 之后选填代码语言。

**解析输出**：
```
<pre>
  <code>
    代码内容
  </code>
</pre>
```

### 3.5 列表

#### 3.5.1 无序列表

Markdown 的无序列表虽然有三种行首标记：加号（`+`）、减号（`-`）还是星号（`*`），但都统一输出为无 `type` 属性的`<ul>` 标签。*朱码*对此做了明确区分：不同的行首标记对应 `<ul>` 标签中不同的 `type` 值。

| 语法       | 描述                    | 解析输出                                 |
| ---------- | ---------------------- | ---------------------------------------- |
| `+ 实心圆` | 行首是加号 + 至少一个空格 | `<ul type="disc"><li>实心圆</li></ul>`   |
| `- 空心圆` | 行首是减号 + 至少一个空格 | `<ul type="circle"><li>空心圆</li></ul>` |
| `* 方块`   | 行首是星号 + 至少一个空格 | `<ul type="square"><li>方块</li></ul>`   |

---

#### 3.5.2 有序列表

Markdown 只支持以阿拉伯数字 + `.` 为行首的有序列表，对应无 `type` 属性的 `<ol>` 标签；*朱码*则扩展到多达 5 种不同 `type` 值的有序列表。

| 语法                  | 描述                                     | 解析输出                                                     |
| --------------------- | ---------------------------------------- | ---------------------------------------------------------- |
| `【!】1. 阿拉伯数字`   | 行首是任意数字 + `.` + 至少一个空格        | `<ol【 reversed】><li>阿拉伯数字</li></ol>`（默认 `type="1"`）|
| `【!】i. 小写罗马数字` | 行首是任意小写罗马数字 + `.` + 至少一个空格 | `<ol type="i"【 reversed】><li>小写罗马数字</li></ol>`       |
| `【!】I. 大写罗马数字` | 行首是任意大写罗马数字 + `.` + 至少一个空格 | `<ol type="I"【 reversed】><li>大写罗马数字</li></ol>`       |
| `【!】a. 小写拉丁字母` | 行首是任意小写字母 + `.` + 至少一个空格     | `<ol type="a"【 reversed】><li>小写拉丁字母</li></ol>`       |
| `【!】A. 大写拉丁字母` | 行首是任意大写字母 + `.` + 至少一个空格     | `<ol type="A"【 reversed】><li>大写拉丁字母</li></ol>`       |

注：
1. 一组列表的第一行的开头数字决定这组列表的起始数字，例如由 `2. 这是第一行` 开始的列表会被解析为 `<ol start="2"><li>这是第一行</li></ol>`。后续行首只要符合对应的数字形式即自动升序，具体写什么数字不是重点，例如在前例后面无论写`3. 这是第二行`还是写`999. 这是第二行`解析输出的结果都是一样的。此功能对罗马数字最友好，即便不熟悉罗马数字的写法，只要列表的每行开头都写为 `i. ` 或 `I. ` 就会自动显示为正常升序的罗马数字。
2. 有序列表的序号也可以按倒序显示，只需在一组列表的第一行行首加上感叹号即可，后续列表行首有没有感叹号都不影响倒序显示。

#### 3.5.3 任务列表

*朱码*延续了 Markdown 的惯例，也提供两种任务列表。

HTML 中并没有直接对应“任务列表”的标签，实现方式是先用 CSS 抹去列表标签默认的行首格式，再借助 `<input type="checkbox">` 来体现任务状态。*朱码*将“任务列表”视为一类特殊的“无序列表”，因此使用三种无序列表的行首标记合理组合为无序列表的行首标记。

| 语法            | 描述                                  | 解析输出                                                                                       | 
| --------------- | ------------------------------------ | --------------------------------------------------------------------------------------------- |
| `+*+ 已完成任务` | 行首是加号 + 星号 + 加号 + 至少一个空格 | `<ul class="task-list"><li><input type="checkbox" checked="" disabled="">已完成任务</li></ul>` |
| `-*- 未完成任务` | 行首是减号 + 星号 + 减号 + 至少一个空格 | `<ul class="task-list"><li><input type="checkbox" disabled="">未完成任务</li></ul>`            |

注：在 `<ul>` 标签中添加了 `class="task-list"` 属性，方便通过 CSS 修改浏览器针对 `<ul>` 的默认样式。

#### 3.5.4 多级嵌套

连续多个行首标记类型相同的行会被解析到同一个 `<ul>` 或 `<ol>` 标签中，每行为一个 `<li>` 标签。仿照 Markdown 的规则，*朱码*同样支持各种列表的多级嵌套，判断层级的依据是行首标记之前缩进的空格数量，例如行首无空格的连续行都位于顶级 `<ul>` 或 `<ol>` 标签中，行首同为两个空格（或一个 TAB）的连续行都位于二级 `<ul>` 或 `<ol>` 标签中。以下是一个多级嵌套不同类型列表的例子。

**原文**：
```
1. 首批任务完成状态
    +*+ 第一项
    -*- 第二项
2. 第二批任务完成状态
    +*+ 第一项
    -*- 第二项
```

**解析输出**：
```html
<ol>
    <li>首批任务完成状态
        <ul class="task-list">
            <li><input type="checkbox" checked="" disabled=""> 第一项</li>
            <li><input type="checkbox" disabled=""> 第二项</li>
        </ul>
    </li>
    <li>第二批任务完成状态
        <ul class="task-list">
            <li><input type="checkbox" checked="" disabled=""> 第一项</li>
            <li><input type="checkbox" disabled=""> 第二项</li>
        </ul>
    </li>
</ol>
```

### 3.6 表格

**语法**：
```
| 表头1 | 表头2 |
| ==== | ==== |
| 行1列1 | 行1列2 |
| ==== | ==== |
| 表尾1 | 表尾2 |
```

**详述**：
1. 当某行的行首和行尾均为竖线（前后允许有空格）时，此行即被识别为表格中的一行。
2. 连续多行符合表格行格式的段落中，行首是“竖线 + 至少4个连续等号 + 竖线”（`|====|`）的行被定义为表格的“结构划分标记行”（以下简称“结构行”） ，用于划分表头（`<thead>`）、表体（`<tbody>`）和表尾（`<tfoot>`）三个部分。
3. 因为“结构行”只起划分表格结构的作用，所以不要求其具备与表格内容行相同的单元格数量，且该行中除了属性章节中的语法以外，任何文字内容均在解析时被丢弃。当需要将某些属性应用到整列时，最好保证“结构行”的单元格数量跟其它行保持一致。
4. 对“结构行”的数量做如下规定：
  * 当某个表格段落中只有**1个**“结构行”时，该表格无表尾 `<tfoot>` 部分，“结构行”之前是表头 `<thead>` 部分，之后是表体 `<tbody>` 部分；
  * 当某个表格段落中符合“结构行”格式的行**不少于2个**时，只有**第1个**和**最后1个**被识别为真正的“结构行”，其余行均按表格内容行处理。第1个“结构行”之前是表头 `<thead>` 部分，最后1个“结构行”后是表尾 `<tfoot>` 部分，两个“结构行”之间是表体 `<tbody>` 部分；
  * 当某个表格段落中完全没有“结构行”时，则整表只含 `<tbody>` 部分。
5. 不论是表头（`<thead>`）、表体（`<tbody>`）还是表尾（`<tfoot>`）部分，均支持任意多个内容行。

**解析输出**：
```html
<table>
  <thead>
    <tr>
      <th>表头1</th>
      <th>表头2</th>
    </tr>
  <tbody>
    <tr>
      <td>行1列1<td>
      <td>行1列2<td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <th>表尾1</th>
      <th>表尾2</th>
    </tr>
  </tfoot>
</table>
```

**单元格跨列与跨行**

| 语法         | 描述                                    | 解析输出            |
| ----------- | ---------------------------------------- | ------------------ |
| `&<正整数>&` | 和号 + 左尖括号 + 正整数 + 右尖括号 + 和号 | ` colspan="正整数"` |
| `&/正整数/&` | 和号 + 斜杠 + 正整数 + 斜杠 + 和号        | ` rowspan="正整数"` |

注：跨行和跨列语法均必须出现在单元格语法范畴内才生效，即必须被竖线所包夹且对“结构行”无效，与单元格内的文字可任意混排，请看如下两则示例。

跨列示例：
```
| 表头1 | 表头2 |
| ==== | ==== |
| 行1列1 &<2>& |
| 行2列1 | 行2列2 |
```
→
```
<table>
  <thead>
    <tr>
      <th>表头1</th>
      <th>表头2</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td colspan="2">行1列1</td>
    </tr>
    <tr>
      <td>行2列1</td>
      <td>行2列2</td>
    </tr>
  </tbody>
</table>
```

跨行示例：
```
| 表头1 | 表头2 |
| ==== | ==== |
| 行1列1 &/2/& | 行1列2 |
| 行2列1 |
```
→
```html
<table>
  <thead>
    <tr>
      <th>表头1</th>
      <th>表头2</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td rowspan="2">行1列1</td>
      <td>行1列2</td>
    </tr>
    <tr>
      <td>行2列1</td>
    </tr>
  </tbody>
</table>
```

### 3.7 嵌入

HTML 本身也是纯文本文件，其中并不包含多媒体内容，但可以通过一些特定标签嵌入到页面中。本章语法即针对这些用于嵌入多媒体的标签。

#### 3.7.1 链接

**语法**：`(【显示文字】){URL【=>打开方式】}`

**详述**：
1. 以一对圆括号起始，其中可选填显示文字（不填则直接显示 URL 本身），紧接着是由一对花括号包裹的 URL，URL 之后还支持用等号+右尖括号作为分隔符追加链接的打开方式。
2. 所谓“打开方式”是指 HTML 的 `<a>` 标签的 `target=` 属性；不填写此部分，则默认为当前窗口（`_self`）。`=>` 后面可接受的值及其含义详见下表，不在此范围内的值将连同分隔符一并被丢弃。
3. 分隔符 `=>` 之前允许有空格，以示与 URL 区隔；其后必须紧跟接受值，不允许有空格。

| 接受值          | 含义    | 简写（不区分大小写）|
| -------------- | ------- | ----------------- |
| `_self`（默认） | 当前窗口 | `self` 或 `s`     |
| `_blank`       | 新窗口   | `blank` 或 `b`    |
| `_parent`      | 父窗口   | `parent` 或 `p`   |
| `_top`         | 满窗口   | `top` 或 `t`      |

**解析输出示例**：

| 原文                                  | 解析输出                                                    |
| ------------------------------------ | ----------------------------------------------------------- |
| `(点击这里){https://example.com}`     | `<a href="https://example.com">点击这里</a>`                 |
| `(){https://example.com}`            | `<a href="https://example.com">https://example.com</a>`     |
| `(点击这里){https://example.com =>b}` | `<a href="https://example.com" target="_blank">点击这里</a>` |

注：这看起来很像 Markdown 的链接语法，但又完全不同。*朱码*所做的修改并非刻意凸现差异，而是为了更简单地避免误判特殊情况。Markdown 的链接语法使用圆括号 `()` 包裹URL，然而根据 RFC 3986 规范，圆括号属于合法字符，即以 `)` 结尾的 URL 可以存在，一旦解析器忽略了这种可能性，就会导致某些特定 URL 因丢失结尾的 `)` 而无法访问。虽然这并不算技术难题，但从源头上避免隐患岂不更好？括号是最直观的包裹符号，而三种括号中只有花括号不在 URL 的合法字符之列，尽管现代浏览器也可以接受包含花括号的网址，但遵守 RFC 3986 规范的 URL 毕竟是绝大多数，因此在*朱码*中的各种语法中只使用花括号包裹 URL。花括号之前的部分之所以改为圆括号，是因为*朱码*将方括号用于属性语法了。

#### 3.7.2 行内图片

**语法**：`!(【alt文字】【/指定宽度|指定高度/】){图片URL}`

**详述**：以感叹号 + 一对圆括号起始，其中可选填显示文字（不填则没有 ` alt=` 属性）并支持用斜杠搭配竖线 `/|/` 指定图片的宽度和高度（具体方法详见下表），紧接着是由一对花括号包裹的图片 URL。

| 填写方式             | 含义                              | 解析输出                            |
| ------------------- | --------------------------------- | ---------------------------------- |
| `/正整数1\|正整数2/` | 指定宽度为“正整数1”且高度为“正整数2” | ` width="正整数1" height="正整数2"` |
| `/正整数/`          | 仅指定宽度为“正整数”                | ` width="正整数"`                   |
| `/\|正整数/`        | 仅指定高度为“正整数”                | ` heigth="正整数"`                  |
| 不填写 `/\|/` 部分   | 不指定宽高，按图片实际大小显示       | 无                                  |

注：宽和高两个数值是相互独立的，如果其中一个值不是正整数，则不输出对应的属性，例如 `/abc|200/` 只输出 ` height="200"`、`/200|6.2/` 只输出 ` width="200"`。

**完整示例**：

| 原文                                             | 解析输出                                                                        |
| ------------------------------------------------ | ------------------------------------------------------------------------------ |
| `!(Logo /50\|50/){https://example.com/drewmark-logo.svg}` | `<img src="https://example.com/drewmark-logo.svg" alt="Logo" width="50" height="50" />` |
| `!(缩略图/200/){https://example.com/image.png}`  | `<img src="https://example.com/image.png" alt="缩略图” width="200" />`          |
| `!(){https://example.com/image.png}`             | `<img src="https://example.com/image.png" />`                                  |

---

#### 3.7.3 图片块

**语法**：
```
!!!
【! 图片标题】
!(【alt文字】【/指定宽度|指定高度/】){图片URL}
【! 图片标题】
!(【alt文字】【/指定宽度|指定高度/】){图片URL}
!!!
```

**描述**：由两个行首是至少三个连续感叹号的行所包夹的多行内容。其中以感叹号 + 至少一个空格开头的行被解析为图片标题（`<figcaption>`）。图片标题行支持文本修饰语法；包含行内图片语法的行将去除多余文字，只解析输出相应的图片标签；不符合上述两种情况的行都将被丢弃（不解析输出任何内容）。

**解析输出**：
```html
<figure>
  【<figcaption>图片标题</figcaption>】
  <img src="...">
  【<figcaption>图片标题</figcaption>】
  <img src="...">
</figure>
```

#### 3.7.4 音频

**语法**：`~(【_】【&】【@】【!】){URL1|媒体类型}【{URL2|媒体类型}】`

**详述**：行首为波浪号 + 一对圆括号，其中可选填附加属性（详见下表1），紧接着是由一对花括号包裹的 URL 和以竖线分隔的媒体类型（支持的类型见下表2），支持用多组花括号结构追加多个不同媒体类型的 URL。需要特别说明的是：`<audio>` 标签虽然在 HTML 的定义中是行内元素，但绝大多数情况下不会混杂在一段文字中间，所以*朱码*将其按块级元素处理，即每条音频语法**必须**占据整行，且只要行首标记符合音频语法，则该行中音频语法之外的多余文字都将被丢弃。

表1：附加属性
| 符号 | 名称   | 对应属性    | 默认值   |
| ---- | ----- | ----------- | ------- |
| `_`  | 下划线 | ` controls` | `true`  |
| `&`  | 和号   | ` autoplay` | `false` |
| `@`  | 艾特号 | ` loop`     | `false` |
| `!`  | 感叹号 | ` muted`    | `false` |

注：
1. 上表中只有 `controls` 属性的默认值与其它三项不同，即 `controls` 属性是默认解析输出的，只有要求**不**输出时才需要填入对应的 `_`。其它三项则是默认不输出的，只有填写了相应的符号才会解析输出该属性。
2. 无论是指定宽高还是这四种属性，均无填写顺序的要求，各部分之间有无空格分隔均可。

表2：媒体类型
| 支持的媒体类型     | 对应值（不区分大小写）   | 解析输出              |
| ----------------- | ---------------------- | -------------------- |
| MP3               | `mp3` 或 `mpeg`        | ` type="audio/mpeg"` |
| WAV (PCM)         | `wav` 或 `wave`        | ` type="audio/wav"`  |
| OGG (Vorbis/Opus) | `ogg`                  | ` type="audio/ogg"`  |
| AAC               | `aac`                  | ` type="audio/aac"`  |
| FLAC              | `flac`                 | ` type="audio/flac"` |
| WebM              | `webm`                 | ` type="audio/webm"` |

注：如果未正确填写支持的媒体类型，则该组 `{URL|媒体类型}` 被视为无效且**不**解析输出。如果各组均未正确填写支持的媒体类型，则无法解析为音频 `<audio>` 标签。因为波浪号之后的语法完全符合链接语法，所以实际上会被解析为 `~<a href="...">...</a>`。

**完整示例**

例1（最简洁）： `~(){https://example.com/music/audio1|mp3}` 解析输出为：
```html
<audio controls>
  <source src="https://example.com/music/audio1" type="audio/mpeg">
  Your browser does not support the audio tag.
</audio>
```

例2（较全面）： `~(&@){https://example.com/music/audio1|mp3}{https://example.com/music/audio2|ogg}` 解析输出为：
```html
<audio controls autoplay loop>
  <source src="https://example.com/music/audio1" type="audio/mpeg">
  <source src="https://example.com/music/audio2" type="audio/ogg">
  Your browser does not support the audio tag.
</audio>
```

注：“Your browser does not support the audio tag.”这句是 `<audio>` 标签默认包含的标准提示语。

#### 3.7.5 视频

**语法**：`$(【{封面图URL}】【/指定宽度|指定高度/】【_】【&】【@】【!】){URL1|媒体类型}【{URL2|媒体类型}】`

**详述**：行首为美元号 + 一对圆括号，其中可选填由花括号包裹的封面图网址、指定宽高（同图片语法中指定宽高的方法）以及附加属性（同音频语法中的表1及相关注释），紧接着是由一对花括号包裹的 URL 和以竖线分隔的媒体类型（支持的类型见下表），支持用多组花括号结构追加多个不同媒体类型的 URL。需要特别说明的是：`<video>` 标签虽然在 HTML 的定义中是行内元素，但绝大多数情况下不会混杂在一段文字中间，所以*朱码*将其按块级元素处理，即每条视频语法**必须**占据整行，且只要行首标记符合视频语法，则该行中视频语法之外的多余文字都将被丢弃。

表：媒体类型
| 支持的媒体类型     | 对应值（不区分大小写）   | 解析输出              |
| ----------------- | ---------------------- | -------------------- |
| MP4               | `mp4` 或 `mpeg`        | ` type="video/mp4"`  |
| WebM              | `webm`                 | ` type="video/webm"` |
| OGG (Vorbis/Opus) | `ogg`                  | ` type="video/ogg"`  |
| AV1               | `av1`                  | ` type="video/av1"`  |
| MOV               | `mov`                  | ` type="video/mov"`  |

注：如果未正确填写支持的媒体类型，则该组 `{URL|媒体类型}` 被视为无效且**不**解析输出。如果各组均未正确填写支持的媒体类型，则无法解析为视频 `<video>` 标签；因为美元号之后的语法完全符合链接语法，所以实际上会被解析为 `$<a href="...">...</a>`。

**完整示例**

例1（最简洁）： `$(){https://example.com/video/clip1|mp4}` 解析输出为：
```html
<video controls>
  <source src="https://example.com/video/clip1" type="video/mp4">
  Your browser does not support the video tag.
</video>
```

例2（较全面）： `$({https://example.com/video/poster.jpg}/400|300/&@){https://example.com/video/clip1|mp4}{https://example.com/video/clip2|webm}` 解析输出为：
```html
<video poster="https://example.com/video/poster.jpg" width="400" height="300" controls autoplay loop>
  <source src="https://example.com/video/clip1" type="video/mpeg">
  <source src="https://example.com/video/clip2" type="video/webm">
  Your browser does not support the video tag.
</video>
```

注：“Your browser does not support the video tag.”这句是<video>标签默认包含的标准提示语。

### 3.8 描述

此组别包含两种用于定义或描述的不同标签。

#### 3.8.1 定义列表

**语法**：`;;术语::定义;;`

**详述**：行首和行尾均为双分号的整行，其中由双冒号分隔开前面的名称部分和后面的描述部分。

**解析输出**：
```html
<dl>
  <dt>术语</dt>
  <dd>定义</dd>
</dl>
```

注：无任何间隔（包括空行）的多行定义语法会被解析在同一个 `<dl>` 标签中，例如：
```
;;名称1:: 描述1;;
;;名称2:: 描述2;;
```
解析输出为：
```html
<dl>
  <dt>名称1</dt>
  <dd>描述1</dd>
  <dt>名称2</dt>
  <dd>描述2</dd>
</dl>
```

---

#### 3.8.2 详情块

**语法**：
```
;;;【标题】
多语法内容
;;;【>】
```

**详述**：由两个行首是至少三个连续分号的行所包夹的多行内容；起始标记行的 `;;;` 之后可选填标题（如不填则不同浏览器会补充一个不同的默认词）；终止标记行的 `;;;` 之后还可追加一个右尖括号，表示页面加载时直接展开详情（默认是折叠状态，即只显示标题或浏览器默认词）。由于语法结构的缘故，详情块是唯一一种能够包含全部标签的标签，在其内容部分不仅可以写任何行内语法，也可以写任何块级语法，都会被正常解析。

**解析输出**

```html
<details【 open】>
  【<summary>标题</summary>】
  <p>多语法内容<p>
</details>
```

**多级嵌套**

详情块支持多级嵌套，不同层级的体现类似于列表的嵌套方式，即以行首缩进的空格数为依据，例如：

```
;;; 一级标题
一级内容
  ;;; 二级标题
  二级内容
  ;;;>
;;;
```
解析输出为：
```html
<details>
  <summary>一级标题</summary>
  <p>一级内容</p>
  <details open>
    <summary>二级标题</summary>
    <p>二级内容</p>
  </details>
</details>
```

### 3.9 引用

#### 3.9.1 行内引用

**语法**：`>>引文【{<=URL}】<<`

**详述**：由双右尖括号和双左尖括号所包裹的内容，支持在引文之后选填用花括号包裹的引用出处，即 `{<=URL}` 部分。

**解析输出**：`<q【 cite="URL"】>引文</q>`

#### 3.9.2 引用块 

**语法**：`> 引用内容【{<=URL}】`

**详述**：行首是一个或连续多个（个数表示嵌套的层级）右尖括号 + 至少一个空格的行，支持选填用花括号包裹的引用出处（同行内引用）。每个层级只支持一个引用出处，且必须位于该层级最后一行的行尾。同一层级的连续多行内容会被解析为一个 `<p>` 段落，每行之间用 `<br>` 换行。如需在同一层级拆分多个 `<p>` 段落，须在段落间插入同层级的空行，即以数量匹配的右尖括号为行首、其后有至少一个空格的行。

**解析输出**：`<blockquote【 cite="URL"】><p>引用内容</p></blockquote>`

**多级多行示例**：

原文：
```
> 一级引用
>> 二级引用行1
>> 二级引用行2【{<=URL}】
>>> 三级引用第一段
>>> 
>>> 三级引用第二段
```

解析输出结果：
```html
<blockquote>
  <p>一级引用</p>
  <blockquote【 cite="URL"】>
    <p>二级引用行1<br>二级引用行2</p>
    <blockquote>
      <p>三级引用第一段</p>
      <p>三级引用第二段</p>
    </blockquote>
  </blockquote>
</blockquote>
```

**包含块级元素**：由于引用块是一种单行块级元素，即以每一行的行首标记决定的块级元素，因此其内部不仅能写入普通段落或行内语法，还能写入其它的单行块级元素——列表、音频、视频和定义列表。这几种包含在引用块中的单行块级元素也能正常解析，例如：
```
> 1. 项目一
> A. 项目A
> ~(){https://example.com/music/audio1|mp3}
> $(){https://example.com/video/clip1|mp4}
> ;;术语::定义;;
```
解析输出为：
```html
<blockquote>
  <ol><li>项目一</li></ol>
  <ol type="A"><li>项目A</li></ol>
  <audio controls>
    <source src="https://example.com/music/audio1" type="audio/mpeg">
    Your browser does not support the audio tag.
  </audio>
  <video controls>
    <source src="https://example.com/video/clip1" type="video/mp4">
    Your browser does not support the video tag.
  </video>
  <dl><dt>术语</dt><dd>定义</dd></dt>
</blockquote>
```

### 3.10 进度条

**语法**：`((数值//【总额】))`

**详述**：由两组圆括号包裹的部分，其中必须用双斜杠分为两部分，且两部分均为正数。总额可不填，默认值为 `1`。

**解析输出**：`<progress value="数值" max="总额"></progress>`

### 3.11 水平分隔线（同 Markdown）

**语法**：`---`

**详述**：行首是至少三个连续的连字符，其后的任何内容都将被丢弃

**解析输出**：`<hr>`

### 3.12 样式块

**语法**：
```
{{{【media】
  多行 css 代码
}}}
```

**详述**：由行首是至少三个左花括号的行和行首是至少三个右花括号的行所包夹的多行内容，起始标记行中 `{{{` 之后可以选填 `media` 属性。

**解析输出**：
```html
<style【 media="media"】>
  多行 CSS 代码
</style>
```

## 4. 属性语法规则

### 4.1 专用属性总结

专用属性与其所属标签语法深度绑定，除了格式以外，填写位置也有一定的要求，在相应的标签语法规则中已有详细讲述，下表仅作为归纳总结，具体用法以对应的标签语法规则为准。

| 属性     | 语法                | 输出                                      | 适用标签范畴            |
| -------- | ------------------ | ----------------------------------------- | ---------------------- |
| 倒序     | `!`                 | ` reversed`                              | 有序列表               |
| 跨列     | `&<正整数>&`        | ` colspan="正整数"`                       | 表格中的内容单元格      |
| 跨行     | `&/正整数/&`        | ` rowspan="正整数"`                       | 表格中的内容单元格      |
| 打开方式 | `=>目标窗口`         | ` target="目标窗口"`                      | 链接                   |
| 指定宽高 | `/正整数1\|正整数2/` | ` width="正整数1" height="正整数2"`       | 图片、视频              |
| 控制等   | `_`/`&`/`@`/`! `   | ` controls`/` autoplay`/` loop`/` muted`  | 音频、视频              |
| 封面图   | `{URL}`            | ` poster="URL"`                           | 视频                   |
| 引用出处 | `<=URL`            | ` cite="URL"`                             | 引用（行内引用和引用块） |
| 展开     | `>`                | ` open`                                   | 详情块                 |
| 媒体类型 | {{{`meida type`    | ` media="media type"`                     | 样式块                 |

### 4.2 全局属性

#### 4.2.1 语法规则

全局属性，顾名思义就是适用于几乎所有标签的属性，所以不像专用属性那样与所属元素的语法深度绑定，而是采用统一的语法规则。

**语法**：`[属性缩写字母[ 属性值 ]【属性缩写字母】]` 

**描述**：左方括号 + 属性缩写字母 + 左方括号 + *属性值* + 右方括号 + 【属性缩写字母】 + 右方括号，其中“属性缩写字母”不区分大小写且第二个可省略。属性值两侧允许有空格，但其余部分不允许有空格。各种全局属性的详细规则如下表：

| 属性     | 语法                                    | 接受值                                               | 输出                                        |
| -------- | --------------------------------------- | -------------------------------------------------- | -------------------------------------------- |
| 水平对齐  | `[A[ 接受值 ]A]`                        | `left`（默认）或 `center` 或 `right`                 | ` align="接受值"`                            |
| 类       | `[C[ 类名 ]C]`                          | 大小写字母、数字、-、_、空格组成的字符串                | ` class="类名"`                             |
| 文字方向  | `[D[ 接受值 ]D]`                        | `ltr`（默认）或 `rtl` 或 `auto`                      | ` dir="接受值"`                              |
| id       | `[I[ ID名 ]I]`                          | 大小写字母、数字、-、_、:、.组成的字符串且必须以字母开头 | ` id="接受值"`                              |
| 语言     | `[L[ 语言名 ]L]`                         | BCP 47标准的语言代码                                 | ` lang="语言名"`                             | 
| 样式     | `[S[ CSS ]S]`                           | CSS样式格式                                          | ` style="样式"`                              |
| 提示     | `[T[ 提示文本 ]T]`                       | 除<、>、&、"以外任意字符组成的字符串                   | ` title="提示文本"`                          |
| data-*   | `[-[ key:接受值【, key2:接受值2】...]-]` | 同上                                                 | ` data-key="接受值"【 data-key2="接受值2"】` |

---

#### 4.2.2 位置与归属

任何标签语法都有其框定的范围，或者是两端标记符之间的部分，或者是符合语法规范的整行。多数情况下，全局属性可以写在某个标签语法框定范围内的任意位置，但也有要求更为严格的情况。

+ **完全不支持**：
  - *表情*：表情语法的解析结果只是一个符号，并不包含任何标签，所以写在表情语法内部的全局属性不仅会原样输出，还会造成无法匹配到所需表情符号。因此，请勿在表情语法框定范围内书写任何全局属性！
  - *行内代码*：`<code>` 标签用于呈现原始代码，所以在行内代码范围内写入全局属性也只会原样输出。

+ **必须写在固定位置**：
  - *代码块*：代码块的内容行跟行内代码的情况一样——所有内容（包括全局属性语法）都会原样输出。不过作为多行的块级元素，代码块的起始标记行除了允许指明代码语言之外，还支持全局属性语法。
  - *样式块*：样式块的形制跟代码块类似，其起始标记行同样支持写入全局属性。
  - *进度条*：因为 `<progress>` 标签要以图形形式呈现 `value` 和 `max` 的百分比关系，所以其语法中相对应的两个部分都必须是正数。因此，进度条语法中的属性必须写在分隔符 `//` 之中，例如：`((32/[C[ green ]C]/46))`，这样才能被解析为 `<progress class="green" value="32" max=“46”></progress>`。
  - *嵌入组*：除图片块以外，嵌入组的四种语法（链接、图片、音频和视频）共享基础结构——`()` + `{}`，其中花括号包裹部分主要用于嵌入的 URL，各种专用属性都写在圆括号包裹部分，顺理成章，全局属性也必须写在 `()` 中。
  - *水平分隔线*：虽然大多数全局属性对于水平分隔线毫无意义，但从 HTML 规范来说，`<hr>` 标签确实支持全局属性，因此*朱码*也支持在其连续多个连字符之后写入全局属性。
  
+ **不同位置指向不同标签**：
  - *音标*：`{{此处的全局属性归属 <ruby> 标签^此处的全局属性归属 <rt> 标签}}`
  - *定义列表*：`;;此处的全局属性归属 <dt> 标签::此处的全局属性归属 <dd> 标签;;此处的全局属性归属 <dl> 标签`。例如：`;;术语 [C[ dt-class ]C]::定义 [C[ dd-class ]C];; [C[ dl-class ]C]` 被解析输出为 `<dl class="dl-class"><dt class=“dt-class”>术语</dt><dd class="dd-class">定义</dd></dl>`。
  - *图片块*：
  ```
  !!!此处的全局属性归属 <figure> 标签
    ！此处的全局属性归属 <figcaption> 标签
    图片语法的全局属性必须写在 `()` 中
  !!!
  ```
  * *详情块*：
  ```
  ;;;此处的全局属性归属 <summary> 标签
    无任何块级元素语法的行，其中的全局属性归属于 <p> 标签
  ;;;
  ```
  - *表格*：结构复杂，详见“父标签属性”中的相关章节。

多重标签语法嵌套时，全局属性归属于所处位置的最内层标签，例如以下三个示例都包含三层标签（`<h1>` → `<strong>` → `<mark>`），全局属性书写在不同位置会归属于不同层级的标签。

| *朱码*内容                                | 全局属性位于    | 全局属性归属标签 | 解析结果                                                                  |
| ----------------------------------------- | ------------- | --------------- | ------------------------------------------------------------------------ |
| `# 这是个**!!重要[C[ important ]]!!**标题` | `!! !!` 范围内 | `<mark>`        | `<h1>这是个<strong><mark class="important">重要</mark></strong>标题</h1>` |
| `# 这是个**!!重要!![C[ important ]]**标题` | `** **` 范围内 | `<strong>`      | `<h1>这是个<strong class="important"><mark>重要</mark></strong>标题</h1>` |
| `# 这是个**!!重要!!**标题[C[ important ]]` | `# ` 行内      | `<h1>`          | `<h1 class="important">这是个<strong><mark>重要</mark></strong>标题</h1>` |

---

#### 4.2.3 非标属性

* 在链接语法中写入语言属性（`([L[ en-US ]L]`）时，其属性名**不是** `lang`，而是 `hreflang`（`<a href="..." hreflang="en-US">...`）。

### 4.3 父标签属性

部分块级元素包含多层标签结构，按照最内层原则，全局属性归属于其所在位置对应的最内层标签。要想达到向不同层级的标签中添加属性的目的，有时需要用到父标签属性。除了以下指明的标签语法之外，其余标签语法并不支持父标签语法，即其中包含的父标签属性语法部分将被丢弃，不会在解析结果中出现。

**语法**：`[属性缩写< 属性值 >【属性缩写】]`

**描述**：简而言之，就是将标准属性语法中双层中括号的内层替换为一对尖括号，其余部分与标准属性语法保持一致。

#### 4.3.1 代码块

代码块的 HTML 结构分为两层：`<pre>` → `<code>`。因为代码块中的全部文字都将原样输出，包括任何 DremMark 语法，所以属性语法必须写在起始标记行之中，标准属性归属于 `<code>`，父标签属性则提升到 `<pre>` 上。例如：

**标准属性**：
````
``` 语言 [C[ code_class ]C]
代码
```
````
→ `<pre><code class="code_class">代码</code></pre>`。

**父标签属性**：
````
``` 语言 [C< pre_class >C]
代码
```
````
→ `<pre class="pre_class"><code>代码</code></pre>`。

---

#### 4.3.2 列表

列表的 HTML 结构分为两层：`<ul>`/`<ol>` → `<li>`，标准属性归属于 `<li>`，父标签属性则提升到 `<ul>`/`<ol>` 上。例如：

**标准属性**：`1. 项目一[A[ center ]A]` → `<ol><li align="center">项目一</li></ol>`，效果是只居中“项目一”。

**父标签属性**：`1. 项目一[A< center >A]` → `<ol align="center"><li>项目一</li></ol>`，效果是整个有序列表都居中。

---

#### 4.3.3 引用块

引用块的 HTML 结构分为两层：`<blockquote>` 和 `<p>`，标准属性归属于 `<p>`，父标签属性则提升到 `<blockquote>` 上。例如：

**标准属性**：`> 第一段 [A[ center ]A]` → `<blockquote><p align=“center”>第一段</p></blockquote>`，效果是只居中这一段。

**父标签属性**：`> 第一段 [A< center >A]` → `<blockquote align="center"><p>第一段</p></blockquote>`，效果是居中整个引用块。

---

#### 4.3.4 详情块

详情块的 HTML 结构分为两层：`<details>` 和 `<summary>`/`<p>`，标准属性按其所处位置归属于对应的 `<summary>`或 `<p>`，父标签属性则提升到 `<details>` 上。例如：

**标准属性**：
```
;;; 标题 [A[ center ]A]
内容
;;;
```
→ `<details><summary align="center">标题</summary><p>内容<p></details>`，效果是只居中“标题”部分。
```
;;; 标题
内容 [A[ center ]A]
;;;
```
→ `<details><summary>标题</summary><p align="center">内容<p></details>`，效果是只居中“内容”部分。

**父标签属性**：
```
;;; 标题 [A< center >A]
内容
;;;
```
或
```
;;; 标题
内容 [A< center >A]
;;;
```
→ `<details align="center"><summary>标题</summary><p>内容<p></details>`，效果是整个详情块都居中。

---

#### 4.3.5 表格

表格的 HTML 结构最为复杂，总共包含四层两层：
1. `<table>`
2. `<thead>`/`<tbody>`/`<tfoot>`
3. `<tr>`
4. `<th>`/`<td>`
根据标准属性和父标签属性的不同位置来决定归属于哪个层级的标签。

**标准属性**：
1. 内容行单元格内的属性都归属于对应的 `<th>`/`<td>`；
2. 内容行行尾处的属性归属于对应该行的 `<tr>`；（注：行尾指的是每行最后一个竖线 `|` 之后的位置，此处的属性之后不再有竖线 `|`）
3. 分隔行单元格内的属性归属于对应该列的每个 `<th>`/`<td>`；
4. 分隔行行尾处的属性归属于 `<table>`。
```
| 对应 <th>                   |  对应 <tr>
| ===== 对应该列所有 <th>/<td> | 对应 <table>
| 对应 <td>                   | 对应 <tr>
| ===== 对应该列所有 <th>/<td> | 对应 <table>
| 对应 <td>                   |  对应 <tr>
```

**父标签属性**：
1. 内容行单元格内的父标签属性都归属于对应该行的 `<tr>`（等同于在该行行尾处的标准属性）；
2. 内容行行尾处的父标签属性归属于对应该部分的 `<thead>`/`<tbody>`/`<tfoot>`；
3. 分隔行单元格内的父标签属性等同于标准属性；
4. 分隔行行尾处的标题属性的父标签写法(`[T< 表格标题 >T]`)将被解析为 `<caption>表格标题</caption>`，其余属性的父标签写法归属于 `<table>`（等同于标准属性）。
```
| 对应 <tr>        | 对应 <thead>
| ===== 同标准属性 | 对应 <table>（标题属性生成 `<caption>` 标签）
| 对应 <tr>        | 对应 <tbody>
| ===== 同标准属性 | 对应 <table>
| 对应 <tr>        | 对应 <tfoot>
```

---

### 4.4 同属性冲突

所谓同属性冲突是指多个相同属性归属于同一个标签的情况，既包括在同一标签范围内重复写了多个相同的标准属性，也包括不同位置的的父标签属性指向同一个父级标签。下面举两个典型的例子，例1对应前者，例2对应后者。
例1：`这是一个普通段落，[A[ center ]A]此处出现了居中属性，结尾处又出现了右对齐属性。[A[ right ]A]`。
例2：
```
1. 项目一中包含让父标签居中的语法 [A< center >A]；
2. 项目二中包含让父标签右对齐的语法 [A< right >A]。
```

按照 HTML 的属性规范，任何标签中，相同属性只能出现一次，因此，发生同属性冲突时只有最后一个生效，之前的将被覆盖。据此，上述例1和例2均只保留右对齐的那条属性，即在相应标签中只解析输出 ` align="right"`。

不过，在 HTML 的全局属性中，类（class）属性与众不同，只有它支持多个值并存，所以类属性冲突的处理方式也区别于其它属性的覆盖原则，而是追加原则。将上述两个例子中的属性缩写字母从 A 改为 C，则会解析输出为 `class="center right"`。

## 5. 转义规则

*朱码*语法用到了所有能够直接从键盘输入的符号。当用户需要书写的内容碰巧与*朱码*的某个语法完全一致时，首选的解决办法是使用代码语法包裹相关内容，即可输出符合*朱码*语法的原文。如果也不适合使用代码语法，或者代码语法本身就是不需要解析的内容，那么就需要使用转义规则。转义规则相当简单，只要在任意*朱码*语法的标记符号前添加反斜杠 `\` 即可，例如：`\**粗体\**`，`*\*粗体\**`，`*\*粗体\**` 或 `*\*粗体*\*` 都不会解析为 `<strong>粗体</strong>`，而是直接原文输出：`**粗体**`；再例如必须要用反引号包裹文字又不希望输出为 `<code>` 标签，只需写作：`` \`不是代码\` ``即可。

使用转义规则有几个注意事项：
1. 如上述两个例子所示，对于包裹型语法应该同时转义其前后标记中至少一个标记符号，因为如果只转义前后标记之一，而前后文中又用到了该语法，那么未转义的标记会匹配到其它同类型语法标记造成非预期的解析结果。此注意事项不仅适用行内标签，代码块、图片块、详情块、样式块等块级标签也属于包裹型语法，区别只在于以整行作为起始和终止标记，转义时也要同时转义起始标记行和终止标记行。
2. 链接语法是两个包裹结构的结合，却比包裹型语法更容易转义，只需转义左圆括号（`\(){URL}`）或左花括号（`()\{URL}`）即可，不会与前后文的其它链接语法发生错误关联。行内图片、音频和视频语法都基于链接语法，主要区别只是前置了不同的符号。如果只转义这三种语法的前置符号，即写作 `\~(){URL}`、`\!(){URL}` 或 `\$(){URL}`，其后的 `(){}` 部分仍然符合链接语法，所以会被解析为相应的前置符号 + `<a>` 标签。正确转义这三种语法的方式与转义链接语法相同，即转义左圆括号或左花括号。
3. 转义引用块、列表等依赖行首标记的语法，只需在其行首标记前添加反斜杠即可。此方法对表格也有效，只不过转义了表格的某个内容行并不影响其它行仍然符合表格语法；要想转义整个表格，则须在其所有行之前添加反斜杠。
4. 在图片块中无法转义行内图片语法，因为图片块中只支持两种类型的行：感叹号 + 空格开头的行对应 `<figcaption>` 标签；包含行内图片语法的行只解析其中的行内图片部分，其余内容被丢弃。一旦转义了图片块中的行内图片，这部分转义内容也归于被丢弃的范畴。

作为转义符的反斜杠 `\` 只要不位于行尾都无法显示。如果必须在行中显示反斜杠，也要对它进行转义：每两个连续的反斜杠会显示为一个反斜杠，即 `\\` 解析为 `\`。

### 5.1 转义无效的特例

前文提到在行内代码或代码块的内部书写任何*朱码*语法的标记符号都会原样输出，但仍有一个符号例外——反引号（`` ` ``），因为反引号是行内代码和代码块自身用到的标记符号。如需在行内引用中显示反引号或者在引用块内的行首位置显示连续三个或更多反引号，使用反斜杠转义并不起作用，因为反斜杠在这两种语法中反倒会被原样输出，无法被视为转义符。遇到这种特例，必须通过反引号的数量来实现，即包裹标记的反引号数量要大于所需显示的反引号数量。

## 6. SVG Logo 使用说明

本项目提供了一个可控制的SVG Logo：`images/drewmark-logo.svg`，其默认宽度为 300px、高度 250px。

### 6.1 调整整体尺寸

通过 CSS 控制 `.drewmark-logo` 的宽度，高度设为自动按比例计算。

```css
/* 固定尺寸 */
.drewmark-logo {
  width: 120px;
  height: auto;       /* ⚠️ 必须保留，否则高度不会自动计算 */
  display: block;     /* 消除 inline 元素底部多余间距 */
}

/* 响应式尺寸 */
@media (max-width: 768px) {
  .drewmark-logo {
    width: 80px;
  }
}
```

### 6.2 两种配色

+ *朱码*的 SVG Logo 默认适配浅色背景，由 `#000000` 和 `#534ab7` 双色组成；当遇到深色模式时自动切换为反色显示。

+ 如需在非深色模式的深色背景上显示反色的*朱码* SVG Logo，请使用 `<object>` 标签引用其 `dark` 主题：`<object data="path/to/drewmark-logo.svg?theme=dark" type="image/svg+xml" width="32px" height="25px"></object>`。注意：宽和高都要设定！
